
LOG4J:

El archivo log4j.xml se utiliza para configurar el logger de la aplicación
Se coloca en la carpeta resources el cual al compilar se coloca en el classpath de la aplicación
no necesita una configuracion adicional para que levante el archivo ya que el pluggin lo levanta del classpath directamente

HIBERNATE:
 
 El archivo hibernate.cfg.xml es el archivo donde se realizan las configuraciones para que hibernate funcione
 al igual que el archivo de log4j.xml se coloca en la carpeta resources para que se incluya en el classpath

 
<hibernate-configuration> este tag le indica a hibernate que de aqui debe tomar las configuraciones para su funcionamiento

<session-factory> le indica a hibernate como configurar la creacion de la sesion de trabajo

<property name="hibernate.connection.driver_class">com.mysql.jdbc.Driver</property> indica que driver va a utilizar hibernate para conectarce a la base de datos

<property name="hibernate.connection.url">jdbc:mysql://localhost:3306/test</property> indica la url donde va a conectarce hibernate

<property name="hibernate.connection.username">root</property> indica con que usuario se va a loguear en la base de datos

<property name="hibernate.connection.password">root</property> indica la contraseña con la que se conectara en la base de datos

<property name="hibernate.dialect">org.hibernate.dialect.MySQLDialect</property> indica el dialecto con el que realizara las consultas a la base de datos , los dialectos disponibles estan el la pag de hibernate.

<property name="show_sql">true</property> se utiliza para mostrar por consola las sentencias sql que envia a la base de datos(solo conviene activarla en etapas de desarrollo ya que todo se imprime en el log)

<property name="hbm2ddl.auto">update</property> esta opcion solo debe utilizarce cuando se desarrolla, ya que lo que hace es decirle a hibernate de que manera va a trabajar con la base de datos el valor update le
 indica que solo va a actualizar la base de datos. existen otras opciones que estan detalladas en la pagina de hibernate

<mapping class="entity.Persona" /> con este tag se indica que clases son las que estan mapeadas, en class se indica la clase incluyendo toda la ruta del paquete.
